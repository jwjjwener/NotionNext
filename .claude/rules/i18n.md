# 国际化（i18n）维护规则

## 架构概览

```
themes/starter/
├── hooks/useStarterI18n.js    # 翻译 hook（核心）
├── components/LangSwitchButton.js  # 语言切换按钮（EN → RO → CN 循环）
├── config.js                  # 三语文案（英文主值 + _CN/_RO 后缀）
└── components/*.js            # 所有组件使用 useStarterI18n
```

**支持语言**：EN（英文，默认）、RO（罗马尼亚语）、CN（中文）

**原理**：`useStarterI18n(key)` 根据 `useGlobal().lang` 判断语言：
- `zh-*` → 返回 `siteConfig(key + '_CN')`
- `ro-*` → 返回 `siteConfig(key + '_RO')`
- 其他（默认英文）→ 返回 `siteConfig(key)`（主配置值）

## 强制规则（CRITICAL）

### 修改英文文案时，必须同步修改中文和罗马尼亚语翻译

当修改 `config.js` 中任何 `STARTER_XXX` 的英文主值时，**必须同时更新对应的 `_CN` 和 `_RO` 翻译**。

```js
// 例：修改定价
STARTER_PRICING_1_PRICE_PERIOD: '/ 60 min',       // ← 改了英文主值
STARTER_PRICING_1_PRICE_PERIOD_CN: '/ 60分钟',     // ← 必须同步改中文
STARTER_PRICING_1_PRICE_PERIOD_RO: '/ 60 min',     // ← 必须同步改罗马尼亚语
```

### 新增配置项时，必须添加 `_CN` 和 `_RO` 翻译

在 `config.js` 中新增任何用户可见的文本配置时：
1. 添加英文主值：`STARTER_XXX_NEW_FIELD: 'English text'`
2. 紧跟添加中文：`STARTER_XXX_NEW_FIELD_CN: '中文文案'`
3. 紧跟添加罗马尼亚语：`STARTER_XXX_NEW_FIELD_RO: 'Text în română'`

### 新增组件时，必须使用 `useStarterI18n`

新建的组件中，所有用户可见的文本必须使用 `useStarterI18n` 而非 `siteConfig`：

```jsx
// ✅ 正确
import { useStarterI18n } from '../hooks/useStarterI18n'
{useStarterI18n('STARTER_NEW_SECTION_TITLE')}

// ❌ 错误 - 不支持翻译
import { siteConfig } from '@/lib/config'
{siteConfig('STARTER_NEW_SECTION_TITLE')}
```

**例外**：URL、图片路径、数字、开关（`_ENABLE`）、货币等不需要翻译的配置，继续使用 `siteConfig`。

## 哪些需要翻译，哪些不需要

| 需要翻译（用 `useStarterI18n`） | 不需要翻译（用 `siteConfig`） |
|---|---|
| 标题、副标题 (`_TITLE`, `_TEXT`) | URL 链接 (`_URL`) |
| 按钮文字 (`_BUTTON_TEXT`) | 图片路径 (`_IMAGE`) |
| 描述文案 (`_DESCRIPTION`) | 开关 (`_ENABLE`) |
| 特性列表 (`_FEATURES`) | 价格数字 (`_PRICE`) |
| 问答 (`_QUESTION`, `_ANSWER`) | 货币符号 (`_PRICE_CURRENCY`) |
| 标签 (`_TAG`) | 图标 (`_ICON`) |
| 页脚标语 (`_SLOGAN`) | 数量 (`_COUNT`) |

## 数组配置的翻译方式

`STARTER_TESTIMONIALS_ITEMS`、`STARTER_TEAM_ITEMS`、`STARTER_FOOTER_LINK_GROUP` 是数组，不走 `useStarterI18n`，需要在组件中使用辅助函数 `t()` 处理：

```jsx
const { lang } = useGlobal()

// 根据语言选择数组项中的翻译字段
const t = (item, field) => {
  if (lang?.startsWith('zh') && item[field + '_CN']) return item[field + '_CN']
  if (lang?.startsWith('ro') && item[field + '_RO']) return item[field + '_RO']
  return item[field]
}

// 使用
{t(item, 'STARTER_TEAM_ITEM_DESCRIPTION')}
```

**新增数组项时**，必须同时添加 `_CN` 和 `_RO` 字段：
```js
{
  STARTER_TEAM_ITEM_AVATAR: '/images/starter/team/team-04.png',
  STARTER_TEAM_ITEM_NICKNAME: 'Alice',
  STARTER_TEAM_ITEM_DESCRIPTION: 'Senior Chinese Teacher',        // 英文主值
  STARTER_TEAM_ITEM_DESCRIPTION_CN: '高级中文教师',                 // 中文翻译
  STARTER_TEAM_ITEM_DESCRIPTION_RO: 'Profesoară senioară de chineză' // 罗马尼亚语翻译
}
```

## 常见维护场景

### 场景 1：修改现有文案

1. 在 `config.js` 中找到对应的 `STARTER_XXX` 键
2. 修改英文主值
3. 找到紧邻的 `STARTER_XXX_CN` 键，修改中文值
4. 找到紧邻的 `STARTER_XXX_RO` 键，修改罗马尼亚语值
5. 推送部署

### 场景 2：新增首页区块

1. 在 `config.js` 中添加新区块的所有配置（英文主值 + `_CN` + `_RO`）
2. 创建新组件，import `useStarterI18n`
3. 所有文本用 `useStarterI18n('STARTER_NEW_XXX')` 读取
4. 在 `index.js` 的 `LayoutIndex` 中引入组件

### 场景 3：新增 FAQ 问题

1. 在 `config.js` 中添加：
   ```js
   STARTER_FAQ_5_QUESTION: 'New question?',
   STARTER_FAQ_5_QUESTION_CN: '新问题？',
   STARTER_FAQ_5_QUESTION_RO: 'Întrebare nouă?',
   STARTER_FAQ_5_ANSWER: 'New answer',
   STARTER_FAQ_5_ANSWER_CN: '新回答',
   STARTER_FAQ_5_ANSWER_RO: 'Răspuns nou',
   ```
2. 在 `FAQ.js` 中复制一个问答区块，改为 `STARTER_FAQ_5_*`

### 场景 4：新增团队成员

在 `config.js` 的 `STARTER_TEAM_ITEMS` 数组中添加：
```js
{
  STARTER_TEAM_ITEM_AVATAR: '/images/starter/team/team-04.png',
  STARTER_TEAM_ITEM_NICKNAME: 'Alice',
  STARTER_TEAM_ITEM_DESCRIPTION: 'Senior Chinese Teacher',
  STARTER_TEAM_ITEM_DESCRIPTION_CN: '高级中文教师',
  STARTER_TEAM_ITEM_DESCRIPTION_RO: 'Profesoară senioară de chineză'
}
```

## 翻译检查清单

每次修改 `config.js` 后，检查：
- [ ] 所有新增/修改的英文文案都有对应的 `_CN` 中文版和 `_RO` 罗马尼亚语版
- [ ] 数组项（Testimonials、Team、Footer Link Group）中的文本字段都有 `_CN` 和 `_RO` 版本
- [ ] 新组件中的文本使用了 `useStarterI18n` 而非 `siteConfig`
- [ ] 部署后在线上点击语言切换按钮验证三种语言

## 语言
切换按钮

`LangSwitchButton.js` 实现 EN → RO → CN 三语循环切换：
```js
const langOrder = ['en-US', 'ro-RO', 'zh-CN']
const langLabels = { 'en-US': 'EN', 'ro-RO': 'RO', 'zh-CN': 'CN' }
```

## 不翻译的内容

- **Notion 菜单**：菜单来自 Notion 数据库（`CUSTOM_MENU: true`），代码中无法翻译
- **博客文章内容**：文章来自 Notion，不在前端翻译范围
- **Logo 文字**：`Logo.js` 直接 import CONFIG，不走翻译系统

## 语言持久化

用户语言选择存储在 `localStorage('lang')`，刷新页面后自动恢复。相关代码在 `lib/global.js` 的 `changeLang` 函数和初始化 `useEffect` 中。
