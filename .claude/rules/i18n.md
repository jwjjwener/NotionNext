# 国际化（i18n）维护规则

## 架构概览

```
themes/starter/
├── hooks/useStarterI18n.js    # 翻译 hook（核心）
├── components/LangSwitchButton.js  # 语言切换按钮
├── config.js                  # 中英文文案（_EN 后缀）
└── components/*.js            # 所有组件使用 useStarterI18n
```

**原理**：`useStarterI18n(key)` 根据 `useGlobal().lang` 判断语言，英文时优先返回 `siteConfig(key + '_EN')`，否则返回 `siteConfig(key)`。

## 强制规则（CRITICAL）

### 修改中文文案时，必须同步修改英文翻译

当修改 `config.js` 中任何 `STARTER_XXX` 的中文值时，**必须同时更新对应的 `STARTER_XXX_EN` 英文值**。

```js
// 例：修改定价
STARTER_PRICING_1_PRICE: '35',        // 改了价格
STARTER_PRICING_1_PRICE_PERIOD: '/ 60分钟',  // ← 改了中文
STARTER_PRICING_1_PRICE_PERIOD_EN: '/ 60 min', // ← 必须同步改英文
```

### 新增配置项时，必须添加 `_EN` 翻译

在 `config.js` 中新增任何用户可见的文本配置时：
1. 添加中文值：`STARTER_XXX_NEW_FIELD: '中文文案'`
2. 紧跟添加英文值：`STARTER_XXX_NEW_FIELD_EN: 'English text'`

### 新增组件时，必须使用 `useStarterI18n`

新建的组件中，所有用户可见的文本必须使用 `useStarterI18n` 而非 `siteConfig`：

```jsx
// ✅ 正确
import { useStarterI18n } from '../hooks/useStarterI18n'
{useStarterI18n('STARTER_NEW_SECTION_TITLE')}

// ❌ 错误 - 不支持翻译
import { siteConfig } from '@/lib/config'
{siteConfig('STARTER_NEW_SECTION_TITLE')}
```

**例外**：URL、图片路径、数字、开关（`_ENABLE`）、货币等不需要翻译的配置，继续使用 `siteConfig`。

## 哪些需要翻译，哪些不需要

| 需要翻译（用 `useStarterI18n`） | 不需要翻译（用 `siteConfig`） |
|---|---|
| 标题、副标题 (`_TITLE`, `_TEXT`) | URL 链接 (`_URL`) |
| 按钮文字 (`_BUTTON_TEXT`) | 图片路径 (`_IMAGE`) |
| 描述文案 (`_DESCRIPTION`) | 开关 (`_ENABLE`) |
| 特性列表 (`_FEATURES`) | 价格数字 (`_PRICE`) |
| 问答 (`_QUESTION`, `_ANSWER`) | 货币符号 (`_PRICE_CURRENCY`) |
| 标签 (`_TAG`) | 图标 (`_ICON`) |
| 页脚标语 (`_SLOGAN`) | 数量 (`_COUNT`) |

## 数组配置的翻译方式

`STARTER_TESTIMONIALS_ITEMS`、`STARTER_TEAM_ITEMS`、`STARTER_FOOTER_LINK_GROUP` 是数组，不走 `useStarterI18n`，需要在组件中手动处理：

```jsx
// 在组件中获取 lang
const { lang } = useGlobal()
const isEn = lang?.startsWith('en')

// 数组项中选择翻译字段
{isEn && item.FIELD_EN ? item.FIELD_EN : item.FIELD}
```

**新增数组项时**，必须同时添加 `_EN` 字段：
```js
{
  STARTER_TEAM_ITEM_NICKNAME: 'Alice',
  STARTER_TEAM_ITEM_DESCRIPTION: '高级中文教师',
  STARTER_TEAM_ITEM_DESCRIPTION_EN: 'Senior Chinese Teacher',  // ← 必须添加
}
```

## 常见维护场景

### 场景 1：修改现有文案

1. 在 `config.js` 中找到对应的 `STARTER_XXX` 键
2. 修改中文值
3. 找到紧邻的 `STARTER_XXX_EN` 键，修改英文值
4. 推送部署

### 场景 2：新增首页区块

1. 在 `config.js` 中添加新区块的所有配置（中文 + `_EN` 英文）
2. 创建新组件，import `useStarterI18n`
3. 所有文本用 `useStarterI18n('STARTER_NEW_XXX')` 读取
4. 在 `index.js` 的 `LayoutIndex` 中引入组件

### 场景 3：新增 FAQ 问题

1. 在 `config.js` 中添加：
   ```js
   STARTER_FAQ_5_QUESTION: '新问题？',
   STARTER_FAQ_5_QUESTION_EN: 'New question?',
   STARTER_FAQ_5_ANSWER: '新回答',
   STARTER_FAQ_5_ANSWER_EN: 'New answer',
   ```
2. 在 `FAQ.js` 中复制一个问答区块，改为 `STARTER_FAQ_5_*`

### 场景 4：新增团队成员

在 `config.js` 的 `STARTER_TEAM_ITEMS` 数组中添加：
```js
{
  STARTER_TEAM_ITEM_AVATAR: '/images/starter/team/team-04.png',
  STARTER_TEAM_ITEM_NICKNAME: 'Alice',
  STARTER_TEAM_ITEM_DESCRIPTION: '高级中文教师',
  STARTER_TEAM_ITEM_DESCRIPTION_EN: 'Senior Chinese Teacher'
}
```

## 翻译检查清单

每次修改 `config.js` 后，检查：
- [ ] 所有新增/修改的中文文案都有对应的 `_EN` 英文版
- [ ] 数组项（Testimonials、Team、Footer Link Group）中的文本字段都有 `_EN` 版本
- [ ] 新组件中的文本使用了 `useStarterI18n` 而非 `siteConfig`
- [ ] 部署后在线上点击语言切换按钮验证

## 不翻译的内容

- **Notion 菜单**：菜单来自 Notion 数据库（`CUSTOM_MENU: true`），代码中无法翻译
- **博客文章内容**：文章来自 Notion，不在前端翻译范围
- **Logo 文字**：`Logo.js` 直接 import CONFIG，不走翻译系统

## 语言持久化

用户语言选择存储在 `localStorage('lang')`，刷新页面后自动恢复。相关代码在 `lib/global.js` 的 `changeLang` 函数和初始化 `useEffect` 中。
